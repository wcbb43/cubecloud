## 魔方云是干什么的？两句话说清楚

CubeCloud的业务其实很简单：卖KVM架构的云服务器。机房就两个地方——中国香港和美国洛杉矶。

但就这两个机房，它做了两条产品线：

**Pro系列**走的是CN2 GIA精品线路，三网去程直连（电信CN2 GIA + 联通9929 + 移动CMIN2），回程同样是CN2 GIA，延迟低、速度稳，建站体验拉满。香港Pro用的是AMD EPYC 7B12处理器+NVMe SSD，洛杉矶Pro用的是Intel Gold 6148+SSD阵列。

**Lite系列**走的是国际线路，香港Lite是BGP混合（NTT+CDN77+CMI+HKIX），洛杉矶Lite回程走联通CU4837，带宽给得大（1Gbps-2Gbps起步），价格也更亲民，适合对大陆直连要求没那么高、但需要大流量的场景。

一个走高端精品路线，一个走大带宽性价比路线，定位很清晰。

## Pro还是Lite？该怎么选

很多人第一次看到这四条产品线会有点懵。其实逻辑很简单，就问自己两个问题：

**第一个问题：你的用户主要在哪里？**

如果你的网站或应用主要面向中国大陆用户，那就选Pro系列。CN2 GIA是目前中国电信提供的最高等级国际出口线路，延迟低、丢包少、晚高峰也不怎么抖。再加上联通9929和移动CMIN2的三网覆盖，不管用户是电信、联通还是移动，体验都不会差。

如果你的业务面向海外，或者对大陆直连速度要求不高（比如挂一些海外服务、做外贸站），那Lite系列就够用了，带宽大、流量多、价格便宜。

**第二个问题：你更看重什么——速度还是带宽？**

Pro系列的带宽相对小（香港Pro是50Mbps起步），但线路质量好；Lite系列带宽大（1Gbps-2Gbps起步），但线路等级没那么高。打个比方，Pro像是走高速公路的小汽车，Lite像是走国道的大货车。目的地一样，但体验不同。

简单说：建站用Pro，跑量用Lite。

香港还是洛杉矶？如果你的访客主要在华南和东南亚，香港机房是首选，延迟可以压到20ms以内。如果访客分布比较广，或者预算有限想要更大的带宽和流量，洛杉矶是更均衡的选择。何况洛杉矶的机器还自带免费的金盾CC硬件清洗防护，做网站不用额外花钱买防御。
